Forecast.io Module

Retrieve current weather conditions and forecasts for a location.

In order to use the API, you will need an API Key. You can sign up for one at https://developer.forecast.io/.
Furthermore I recommend to study the API developer docs you can find at https://developer.forecast.io/docs/v2

This module is developed by DevilSquid – your learning resource for Corona SDK.

Please make sure what the Terms of Use of this API service are. This module is a connection to a third party API service to use their data. Have in mind that these services do have their own requirements and attribution guidelines you have to comply with.

ForeCastIO:new (options, forceRequest)
Create a new forecast.io weather object. The request gets send to forecast.io API server. The answer with weather data gets saved to local directory, to prevent too many requests to the API server

Parameters:

  • options

    a table with options

    • apiKey: (string) the api key you get from forecast.io
    • town: (string) the name of the town you want to request weather data for
    • street: (string) the street and house number
    • zip: (string) the zip code as a string
    • country: (string) the full name or short code of the country
    • expires: (number) how long before the weather data expires in seconds (default: 3600)
    • units: (string) metric “si” or imperial “us” units or “ca” or “uk2” (default: “us”)
    • language: (string) short code for the language, see here which languages are supported https://developer.forecast.io/docs/, does not have effect on the units parameter (default: “en”)
    • callback: (function) an outer callback function that gets called, when the data is ready to be used
  • forceRequest bool forces a request to forecast.io’s weather server, although the data has not been expired

Returns:

    forecast.io weather object

Usage:

     local function WeatherListener( event )
      if event.type == "ready" then
          print( "Temperature: ",                 event.forecastio:GetCondition("temp", true) )
          print( "Summary: ",                     event.forecastio:GetCondition("summary", true) )
          print( "Ozone: ",                       event.forecastio:GetCondition("ozone", true) )
          print( "Precipitation Type: ",          event.forecastio:GetCondition("preciptype", true) )
          print( "Precipitation Probability: ",   event.forecastio:GetCondition("precipProbability", true) )
          print( "Pressure: ",                    event.forecastio:GetCondition("pressure", true) )
          print( "Humidity: ",                    event.forecastio:GetCondition("humidity", true) )
      end
 end
 local weather = ForeCastIO:new({
     apiKey      = "YOUR_API_KEY",
     town        = "Barsinghausen",
     street      = "Siehkamp 12",
     zip         = "30890",
     country     = "de",
     units       = "si",
     callback    = WeatherListener
     })
forecast:GetCondition (optDataKind, optWithUnits)
Read the current weather conditions from locally saved weather data

Parameters:

  • optDataKind string what kind of data you want to read
  • optWithUnits bool

    true if units should be added to the return value string, default = false

    You can use the following values for optDataKind:

    • time: The UNIX time (that is, seconds since midnight GMT on 1 Jan 1970) at which this data point occurs.
    • temperature: numerical values representing the current temperatures
    • apparentTemperature: numerical values representing the current apparent (or “feels like”) temperatures
    • precipType: A string representing the type of precipitation occurring at the given time. If defined, this property will have one of the following values: rain, snow, sleet (which applies to each of freezing rain, ice pellets, and “wintery mix”), or hail.
    • precipProbability: A numerical value between 0 and 1 (inclusive) representing the probability of precipitation
    • precipIntensity: A numerical value representing the average expected intensity (in inches of liquid water per hour) of precipitation occurring at the given time conditional on probability (that is, assuming any precipitation occurs at all). A very rough guide is that a value of 0 in./hr. corresponds to no precipitation, 0.002 in./hr. corresponds to very light precipitation, 0.017 in./hr. corresponds to light precipitation, 0.1 in./hr. corresponds to moderate precipitation, and 0.4 in./hr. corresponds to heavy precipitation.
    • windSpeed: A numerical value representing the wind speed in miles per hour
    • windBearing: A numerical value representing the direction that the wind is coming from in degrees, with true north at 0° and progressing clockwise. (If windSpeed is zero, then this value will not be defined.)
    • icon: A machine-readable text summary of this data point, suitable for selecting an icon for display. If defined, this property will have one of the following values: clear-day, clear-night, rain, snow, sleet, wind, fog, cloudy, partly-cloudy-day, or partly-cloudy-night. (Developers should ensure that a sensible default is defined, as additional values, such as hail, thunderstorm, or tornado, may be defined in the future.)
    • pressure: A numerical value representing the sea-level air pressure in millibars.
    • ozone: A numerical value representing the columnar density of total atmospheric ozone at the given time in Dobson units. the following values: rain, snow, sleet (which applies to each of freezing rain, ice pellets, and “wintery mix”), or hail. (If precipIntensity is zero, then this property will not be defined.)
    • cloudCover: A numerical value between 0 and 1 (inclusive) representing the percentage of sky occluded by clouds.
    • humidity: A numerical value between 0 and 1 (inclusive) representing the relative humidity.
    • dewPoint: A numerical value representing the dew point at the given time in degrees Fahrenheit
    • summary: A human-readable text summary of this data point. (Do not use this value for automated purposes: you should use the icon property, instead.)

Returns:

    the value of the requested data

Usage:

     local function WeatherListener( event )
      if event.type == "ready" then
          print( "Temperature", event.forecastio:GetCondition("temperature", true) )
      end
 end
 local weather = ForeCastIO:new({
      apiKey      = "YOUR_API_KEY",
      town        = "Barsinghausen",
      street      = "Siehkamp 12",
      zip         = "30890",
      country     = "de",
      units       = "si",
      callback    = WeatherListener
 })
forecast:GetDailyForecast (optDay, optDataKind, optWithUnits)
Read the daily forecast weather conditions from locally saved weather data

Parameters:

  • optDay number number of days in the future to get the forecast for, 1=tomorrow, 2=day after tomorrow … etc, max number is 8 days
  • optDataKind string the kind of data you want to get
  • optWithUnits bool

    true if units should be added to the temperature forecast values, default = false

    You can use the following values for optDataKind:

    • time: The UNIX time (that is, seconds since midnight GMT on 1 Jan 1970) at which this data point occurs.
    • temperatureMax: numerical values representing the maximumum temperatures on the given day
    • temperatureMin: numerical values representing the minimum temperatures on the given day
    • temperatureMaxTime: UNIX time of the maximumum temperatures
    • temperatureMinTime: UNIX time of the minimum temperatures
    • apparentTemperatureMax: numerical values representing the maximumum apparent (or “feels like”) temperatures
    • apparentTemperatureMin: numerical values representing the minimum apparent (or “feels like”) temperatures
    • apparentTemperatureMaxTime: UNIX time of the maximumum apparent temperatures
    • apparentTemperatureMinTime: UNIX time of the minimum apparent temperatures
    • precipType: A string representing the type of precipitation occurring at the given time. If defined, this property will have one of the following values: rain, snow, sleet (which applies to each of freezing rain, ice pellets, and “wintery mix”), or hail.
    • precipProbability: A numerical value between 0 and 1 (inclusive) representing the probability of precipitation
    • precipIntensity: A numerical value representing the average expected intensity (in inches of liquid water per hour) of precipitation occurring at the given time conditional on probability (that is, assuming any precipitation occurs at all). A very rough guide is that a value of 0 in./hr. corresponds to no precipitation, 0.002 in./hr. corresponds to very light precipitation, 0.017 in./hr. corresponds to light precipitation, 0.1 in./hr. corresponds to moderate precipitation, and 0.4 in./hr. corresponds to heavy precipitation.
    • precipIntensityMax: numerical values representing the maximumum expected intensity of precipitation on the given day in inches of liquid water per hour.
    • precipIntensityMaxTime: the UNIX time of precipIntensityMax
    • windSpeed: A numerical value representing the wind speed in miles per hour
    • windBearing: A numerical value representing the direction that the wind is coming from in degrees, with true north at 0° and progressing clockwise. (If windSpeed is zero, then this value will not be defined.)
    • icon: A machine-readable text summary of this data point, suitable for selecting an icon for display. If defined, this property will have one of the following values: clear-day, clear-night, rain, snow, sleet, wind, fog, cloudy, partly-cloudy-day, or partly-cloudy-night. (Developers should ensure that a sensible default is defined, as additional values, such as hail, thunderstorm, or tornado, may be defined in the future.)
    • pressure: A numerical value representing the sea-level air pressure in millibars.
    • ozone: A numerical value representing the columnar density of total atmospheric ozone at the given time in Dobson units. the following values: rain, snow, sleet (which applies to each of freezing rain, ice pellets, and “wintery mix”), or hail. (If precipIntensity is zero, then this property will not be defined.)
    • cloudCover: A numerical value between 0 and 1 (inclusive) representing the percentage of sky occluded by clouds.
    • humidity: A numerical value between 0 and 1 (inclusive) representing the relative humidity.
    • dewPoint: A numerical value representing the dew point at the given time in degrees Fahrenheit
    • sunsetTime: The UNIX time of the last sunrise before and first sunset after the solar noon closest to local noon on the given day.
    • sunriseTime: The UNIX time of the last sunrise before and first sunset after the solar noon closest to local noon on the given day.
    • summary: A human-readable text summary of this data point. (Do not use this value for automated purposes: you should use the icon property, instead.)
    • moonPhase: A number representing the fractional part of the lunation number of the given day: a value of 0 corresponds to a new moon, 0.25 to a first quarter moon, 0.5 to a full moon, and 0.75 to a last quarter moon. (The ranges in between these represent waxing crescent, waxing gibbous, waning gibbous, and waning crescent moons, respectively.)

Returns:

    the requested forecast value as a string

Usage:

     local function WeatherListener( event )
      if event.type == "ready" then
          print( "Temperature", event.forecastio:GetDailyForecast(2, "temperatureMax", true) )
      end
 end
